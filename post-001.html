欢迎 & 这个博客是怎么搭出来的

WYQ 2026 · 05 · 15 10 分钟阅读

你好, 欢迎来到 WYQ 专属博客。 这是这里的第一篇文章 — 我会先介绍这个博客本身, 然后把从零开始搭建这种静态个人博客的全部步骤详细写下来, 让任何对编程一无所知的朋友, 也能照着做出一个属于自己的、免费、可全球访问的网站。

一、关于这个博客

这个博客是我的个人空间, 不为流量, 不为变现, 只为留下一些可以回头翻阅的文字。 它的设计灵感来自苹果官网 — 大字标题、毛玻璃顶栏、圆角卡片、柔和阴影, 以及那种"留白即奢侈"的克制感。

它有哪些功能

  • 苹果官网风的视觉语言 — SF 字体栈、`#0071e3` 主蓝、巨型 Hero 标题。
  • 暗 / 亮主题切换 — 自动跟随系统, 也可以点右上角图标手动切换, 选择会被记住。
  • 主页只放介绍 — Hero、关于我、个人资料、跳转文章按钮, 不在首页堆文章。
  • 独立的文章详情页 — 苹果风排版, 引言段加大、二级标题清晰、引用块带蓝色侧边线。
  • 完全静态 — 没有数据库, 没有后端, 没有构建步骤, 一份 HTML 改完就能上线。

它用了哪些"技术"

只有最朴素的三样东西: HTML / CSS / JavaScript。 没有 React, 没有 Vue, 没有 Next.js, 也没有 Tailwind。 托管在 GitHub Pages, 永久免费、自带 HTTPS、全球加速。

“最适合你的工具, 是你能完全掌控的那个。”

二、搭建教程: 从零到上线

下面这份教程, 假设你 从未写过代码。我会把每一步都说清楚。 从注册账号到博客上线, 大约需要 30 分钟。

你需要的东西

  • 一台能上网的电脑（Windows / Mac / Linux 都行）
  • 一个邮箱（用来注册 GitHub 账号）
  • 一颗愿意慢慢来的心

第 1 步: 注册一个 GitHub 账号

github.com/signup 用邮箱注册一个账号。 记下你的用户名 — 它会出现在你博客的网址里, 例如 https://你的用户名.github.io/。

完成邮箱验证后, 你就拥有了一个属于自己的 GitHub。这是接下来一切的起点。

第 2 步: 创建一个仓库（Repository）

"仓库"听起来吓人, 其实就是一个云端的项目文件夹, 用来放你博客所有的文件。

  1. 登录后, 点右上角 + → New repository
  2. Repository name 填一个名字, 比如 my-book 或 blog
  3. 选 Public（必须公开, GitHub Pages 才免费）
  4. 勾上 Add a README file
  5. 点 Create repository

创建好之后, 你会进入仓库主页。把它的网址记下来, 类似: 

https://github.com/你的用户名/my-book

第 3 步: 把博客文件放进仓库

如果你想直接用我现在这个博客的代码作为起点 — 仓库 Riuip/my-book 是开源的, 可以参考。

最简单的两种做法:

做法 A: 在 GitHub 网页上手动创建文件

  1. 进你刚创建的仓库
  2. 点 Add file → Create new file
  3. 文件名输入 index.html
  4. 把内容粘贴进去
  5. 滚到底部, 点 Commit changes
  6. 重复以上过程, 把 post-001.html、assets/css/style.css、assets/js/main.js、.nojekyll 都建好

其中 assets/css/style.css 这种带斜杠的文件名, GitHub 会自动帮你创建文件夹。

做法 B: 用本地工具上传（更专业一点）

如果你愿意装个 GitHub Desktop 或者 VS Code, 体验会更舒服:

  1. 用 GitHub Desktop "Clone" 你的仓库到本地
  2. 用 VS Code 打开那个文件夹, 编辑文件
  3. 在 GitHub Desktop 里写一句 commit message, 点 Commit to main, 再点 Push origin

第 4 步: 启用 GitHub Pages

这是把仓库变成网站的关键一步:

  1. 进入仓库的 Settings（设置）
  2. 左侧菜单找到 Pages
  3. Source: 选 Deploy from a branch
  4. Branch: 左边选 main, 右边选 / (root)
  5. 点 Save

等 1 到 2 分钟, 页面顶部会出现一句话:

Your site is live at https://你的用户名.github.io/my-book/

点那个链接 — 那就是你的博客了 🎉

第 5 步: 写一篇属于你的真文章

这是最有趣的一步。打开仓库里的 post-template.html, 这是我准备好的文章模板:

  1. 把它复制一份, 重命名为 post-001.html（或者 post-hello.html 之类的名字）
  2. 修改文件最上面的 <title> 为你的文章标题
  3. 修改 article-hero__title 里的标题
  4. 修改 article-hero__tag 里的分类（比如 "生活" / "思考"）
  5. 修改 article-hero__meta 里的日期和阅读时长
  6. 把 <article class="article"> 标签里的内容换成你自己写的

正文写作的"小语法"

模板支持的元素都在样式里有对应的精致排版:

  • <p class="lead"> — 引言段（开头那段加大字号）
  • <h2>、<h3> — 一级 / 二级小标题
  • <p> — 普通段落
  • <ul> < <li> > — 无序列表
  • <ol> — 有序列表（带数字）
  • <blockquote> — 引用块, 适合放一句金句
  • <code> — 行内代码
  • <pre><code> — 多行代码块
  • <hr/> — 分隔线

第 6 步: 让主页跳转到你的新文章

打开 index.html, 用 Ctrl+F 搜索 post-template.html, 会找到 4 处出现这个文件名 (导航栏一处, 三个"查看文章"按钮各一处)。 把它们都替换成你的新文件名 post-001.html, 保存提交。

等 1 分钟, 刷新主页, 点"查看文章" — 它已经会跳到你的真文章了。

第 7 步: 持续更新

之后每写一篇新文章, 流程都一样:

  1. 复制 post-template.html, 改名（比如 post-002.html）
  2. 填内容
  3. 提交并推送（GitHub Desktop 一键, 或网页直接编辑）
  4. 1 分钟后线上就更新了

三、几个进阶技巧

1. 自定义域名（可选）

如果你想用自己买的域名（比如 wyq.com）, 可以在仓库的 Settings → Pages 里 添加 Custom domain, 再去你买域名的服务商后台加一个 CNAME 解析到 你的用户名.github.io。 GitHub Pages 会自动给你颁发免费的 HTTPS 证书。

2. 改主题色

打开 assets/css/style.css, 找到顶部的: 

--accent: #0071e3;

把这个 16 进制颜色换成你喜欢的, 整站的主蓝就变了。比如:

  • 橘色: #ff6b35
  • 紫色: #8b5cf6
  • 绿色: #10b981
  • 粉色: #ec4899

3. 调字号 / 间距

所有重要的字号都用了 clamp(), 比如: 

.hero__title {
  font-size: clamp(64px, 11vw, 124px);
}

三个值分别是: 最小、随屏幕缩放、最大。把第三个值改大一点, 大屏上的 Hero 就会更"凶"。

4. 备份你的博客

因为所有内容都在 GitHub 上, 它本身就是备份。但建议你偶尔把仓库 Clone 一份到本地硬盘, 就算 GitHub 哪天抽风, 你的文字也永远在你手里。

四、最后想说

建一个博客其实并不难。难的是 — 在这个被算法塞满的时代, 还愿意为自己留一处, 不被打扰的角落。

如果你也想搭一个属于自己的小站, 这份教程对你有用就好。 做好了, 欢迎写信告诉我, 我会很高兴。

“写, 是为了让那些容易被忘记的瞬间, 留下来。”
