Web-Omni: 我做了一个 17 合 1 的浏览器插件

WYQ 2026 · 05 · 15 12 分钟阅读

这一篇, 我想跟你聊聊我自己做的一个开源项目 — Web-Omni (网页全能王)。 它是一个浏览器扩展, 把 17 个我自己最常用的“网页小工具”都揉到了同一个插件里。 下面是它的功能、架构, 以及实现过程中我踩过的一些坑。

一、它是个什么

简单说, 就是一个 Chrome 浏览器扩展。装上以后, 你的浏览器会多出一组以前需要装十几个插件、 或者完全没人做的功能。Manifest V3, 体积不大, 没有任何远程后端, 不收集任何数据 — 所有计算都在你自己的浏览器里发生。

  • 名字: Web-Omni (网页全能王)
  • 当前版本: v1.1 (插件 manifest 内为 4.0.0)
  • 开源协议: GitHub 仓库开源
  • 下载: github.com/Riuip/web-Omni-/releases
“不为做一个完美的产品, 只为把自己最常重复的操作, 变成一次安装。”

二、它能做什么 — 17 个内置功能

Web-Omni 由一个 background.js 后台 + 17 个独立的 content script 组成。 每个脚本对应一个“功能模块”, 互相不依赖, 想用哪个就用哪个。

1. 视觉控制类

  • visual-dictator — 自定义页面字号、字体、行高、颜色
  • immersive-modding — 沉浸阅读模式, 一键去除花花绿绿的干扰
  • sticky-killer — 一键清除粘性导航条、悬浮广告、回到顶部按钮
  • element-pip — 把页面任意元素“摘下来”做成画中画悬浮窗

2. 数据与自动化

  • data-harvester — 一键抓取页面表格 / 列表为 CSV / JSON
  • automation-geek — 录制和回放点击, 用于重复性操作
  • ecommerce-scraper — 电商页商品信息提取
  • price-comparator — 多平台价格比较
  • dom-monitor — 监控页面 DOM 变化, 比如“有货提醒”

3. 隐私与安全

  • privacy-shield — 隐私保护, 拦截常见追踪脚本
  • clean-url — 自动清洗 URL 中的追踪参数 (utm_xxx, fbclid 等)
  • password-vault — 本地密码库 (单独的 vault.html 页面)
  • input-timemachine — 输入历史记录, 防止表单一刷就丢

4. 媒体与传输

  • audio-normalizer — 自动平衡网页里各处音频的音量
  • youtube-enhancer — 给 YouTube 加一些原生没有的功能
  • lan-transfer — 局域网手机↔电脑互传文件 (基于 WebRTC)

5. 工具与中枢

  • page-tools — 页面工具集合
  • command-hub — 快捷指令中枢, Ctrl+Shift+K 呼出, 是所有功能的入口

三、技术架构

1. Manifest V3 + 17 个 content script

用的是 Chrome 推荐的最新插件标准 Manifest V3。 它有几个核心约束:

  • 后台脚本必须是 service_worker, 不能常驻; 用完即休眠
  • 不能用 eval、不能动态注入远程脚本
  • 跨脚本通信必须走 chrome.runtime.sendMessage / onMessage

所有 content script 都注入到 <all_urls> 上, 但默认“沉睡” — 只在用户从 command-hub 主动激活某功能时, 才真正干活。

2. 后台 service worker 是个“消息中继”

background.js 本身没有业务逻辑, 它只做两件事: 

// 1. 把任何 content script 发的消息, 中继给同 tab 的所有脚本
chrome.runtime.onMessage.addListener((req, sender, sendResponse) => {
  if (sender.tab && req.action) {
    chrome.tabs.sendMessage(sender.tab.id, req);
  }
});

// 2. 监听全局快捷键, 触发 command-hub 显示
chrome.commands.onCommand.addListener((command) => {
  if (command === "toggle-command-hub") {
    chrome.tabs.query({ active: true, currentWindow: true }, (tabs) => {
      if (tabs.length) chrome.tabs.sendMessage(tabs[0].id, { action: "TOGGLE_COMMAND_HUB" });
    });
  }
});

这个“轻后台”模式, 让插件冷启动几乎为零开销, 也最大程度避开了 V3 service worker 被频繁回收带来的状态丢失问题。

3. 一切配置都在 chrome.storage

用户在 command-hub 里勾选的开关、自定义字号、密码库的密文等, 全部写到 chrome.storage.local。这有两个好处:

  • 关闭浏览器再打开, 状态原样恢复
  • 未来可以无缝切到 chrome.storage.sync 实现跨设备同步

四、最难的两个模块

1. lan-transfer — WebRTC 局域网传文件

最早这个功能是想要“手机和电脑一键互传” — 后来发现远比想象的复杂。 最终采用了两套模式:

  1. 模式 A: 从插件直接“下载手机页面 HTML”, 用户把 HTML 发给手机, 手机用浏览器打开, 输入房间码连接 — 简单, 但需要 STUN 服务可用
  2. 模式 B: 一键导出“局域网页包 ZIP”, 里面是 index.html + start-mobile-server.cmd + serve-mobile.ps1。 用户在 Windows 上双击运行, 手机直接用 IP 访问, 稳定得多

涉及到的技术栈: WebRTC DataChannel + peerjs + QRCode + 本地 PowerShell HTTP server。

“在内网传一个文件, 居然要一套 WebRTC + STUN + 房间码 — 网络这件事, 越深入越敬畏。”

2. command-hub — 17 合 1 的入口

因为有 17 个功能, 不能让用户右键 17 个菜单, 也不能让浏览器图标弹出一堆开关。 最终的设计是:

  • 全局快捷键 Ctrl+Shift+K 呼出一个浮层
  • 浮层是 Spotlight 风格 — 顶部一个搜索框, 下面是模糊匹配的功能列表
  • 每个功能项右边显示当前状态 (开 / 关 / 配置中)
  • 键盘上下选, Enter 触发, Esc 关闭

为了不污染网页的样式, command-hub 用了 Shadow DOM 包起来, 网页本身的 CSS 完全影响不到它。

五、几个我特别得意的细节

1. 沉浸阅读 — 不只是“清空样式”

市面上的“阅读模式”基本都是“识别文章主体, 重排页面”。我没用现成方案, 而是用了 启发式选择器评分: 给页面所有元素打分 (文字密度、图片比例、链接占比), 最终得分最高的子树就是“正文”。准确率比想象中高很多。

2. 输入时光机 — 防止表单一刷就丢

监听所有 input / textarea 的输入事件, 用 防抖 + IndexedDB 异步保存。 浏览器崩溃后再打开, 自动检测当前 URL 有没有过去的草稿, 提示恢复。 “再也不怕填长表填到最后一刷新就没了”。

3. 价格比较器 — 跨域请求是怎么解决的

在某宝商品页, 怎么去拿京东的价格? 答案是: permissions: <all_urls> + chrome.declarativeNetRequest。借助插件特权, content script 可以发任意跨域请求, 然后把结果回传到当前页。 没有 CORS 之苦。

六、踩过的坑

1. service worker 会被随时杀掉

Manifest V3 的 service worker 不像 V2 的 background page, 它会在 30 秒空闲后被回收。 我一开始把状态存在 self 上, 结果一会儿就丢。最终方案: 所有状态都进 chrome.storage, service worker 每次唤醒时从 storage 重建。

2. content script 的样式污染

把 UI 直接 inject 进网页的话, 网页的 * { margin: 0 } 之类的会破坏一切。 解决: Shadow DOM 隔离 + 内联所有样式。

3. 17 个脚本同时注入会变慢

早期所有脚本 document_start 注入, 页面打开慢得肉眼可见。 改成 document_idle + 每个脚本“懒激活” (只 import 接口, 真要用时再 import 实现) 后, 一切都正常了。

七、想试用？

下载方式:

  1. 访问 GitHub Releases 页面
  2. 下载最新版的 ZIP
  3. 解压
  4. Chrome → 扩展程序 → 打开“开发者模式”
  5. 点“加载已解压的扩展程序”, 选刚才解压的目录
  6. Ctrl+Shift+K 呼出 command-hub, 开始玩

项目完全开源, 欢迎 star、Issue、PR — github.com/Riuip/web-Omni-。
